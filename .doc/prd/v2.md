## 1. 概述

### 1.1 项目背景

HiFiTi 是一款基于 hifiti.com 的 Android 音乐播放应用，当前已上线核心功能：关键词搜索、流式播放、歌曲收藏、本地下载。

当前版本存在以下核心体验缺陷：

| 问题 | 用户影响 | 严重程度 |
|------|----------|----------|
| 无底部导航栏，页面切换依赖路由跳转 | 用户无法在核心模块间快速切换，操作路径长 | 高 |
| 无播放历史 | 用户无法找回「听过但忘记名字」的歌曲 | 高 |
| 播放器无歌单概念 | 每次只能播放单曲，无法连续听歌 | 高 |
| 无播放模式控制 | 用户无法按偏好方式消费内容 | 中 |

### 1.2 产品目标

通过本次迭代，完成从「单曲工具」到「个人音乐播放器」的体验升级：

- **效率提升**：底部导航让核心操作一触即达
- **记忆沉淀**：播放历史帮助用户积累个人音乐库
- **连续体验**：歌单播放让用户沉浸式听歌，无需反复手动操作
- **个性控制**：播放模式满足不同场景的收听偏好

### 1.3 成功指标

| 指标 | 目标 |
|------|------|
| 「我的」页面日均访问率 | > 40% DAU |
| 人均单次连续播放歌曲数 | 从 1 首提升至 3+ 首 |
| 播放历史日均使用率 | > 20% DAU |

## 3. 功能详细设计

### F1. 底部导航栏

**优先级**：P0 | **复杂度**：中

#### 3.1.1 需求描述

在 App 底部新增常驻导航栏，为用户提供一级页面的快速切换能力。

#### 3.1.2 导航项定义

| 序号 | 标签 | 图标（Material Icons） | 目标页面 |
|------|------|----------------------|----------|
| 1 | 首页 | `Icons.Default.Home` | 搜索页（现有） |
| 2 | 我的 | `Icons.Default.Person` | 个人中心页（新增） |

#### 3.1.3 布局结构

当前结构：

```
┌─────────────────────────┐
│       页面内容            │
├─────────────────────────┤
│     Mini 播放器          │  ← Scaffold bottomBar
└─────────────────────────┘
```

目标结构：

```
┌─────────────────────────┐
│       页面内容            │
├─────────────────────────┤
│     Mini 播放器          │  ← 有歌曲播放时显示
├─────────────────────────┤
│   首页    |    我的       │  ← 底部导航栏（常驻）
└─────────────────────────┘
```

#### 3.1.4 交互规则

| 规则 | 说明 |
|------|------|
| **Tab 切换** | 点击 Tab 切换对应页面，选中态高亮（图标 + 文字变色） |
| **状态保持** | 切换 Tab 时保留各页面状态（搜索结果、滚动位置等），不销毁页面 |
| **详情页处理** | 进入歌曲详情页时，隐藏底部导航栏（仅保留 Mini 播放器）；返回时恢复导航栏，回到之前选中的 Tab |
| **Mini 播放器关系** | Mini 播放器位于导航栏上方，两者独立显示/隐藏 |
| **默认选中** | App 启动默认选中「首页」Tab |

#### 3.1.5 验收标准

- [ ] 底部导航栏常驻显示「首页」「我的」两个 Tab
- [ ] 点击 Tab 切换页面，选中态有明确视觉反馈
- [ ] Tab 切换不丢失页面状态
- [ ] 详情页隐藏导航栏，返回后恢复
- [ ] Mini 播放器位于导航栏上方，互不干扰

### F2.「我的」页面 — 收藏歌曲

**优先级**：P0 | **复杂度**：中

#### 3.2.1 需求描述

新增「我的」页面，将现有收藏功能从独立页面迁移至此，与播放历史共同构成用户的个人音乐空间。

#### 3.2.2 页面结构

「我的」页面顶部使用 Tab 页签切换两个子列表：

```
┌─────────────────────────┐
│  我的                    │  ← TopAppBar
├────────────┬────────────┤
│  收藏歌曲  │  播放历史   │  ← Tab 页签（左右滑动切换）
├────────────┴────────────┤
│  ▶ 播放全部    共 N 首   │  ← 操作栏
├─────────────────────────┤
│  🎵 歌曲名              │
│     歌手名               │  ← 歌曲列表项
├─────────────────────────┤
│  🎵 歌曲名              │
│     歌手名               │
├─────────────────────────┤
│         ...              │
└─────────────────────────┘
```

#### 3.2.3 收藏歌曲 Tab

**列表展示**

| 元素 | 说明 |
|------|------|
| 操作栏 | 左侧「播放全部」按钮，右侧显示歌曲总数「共 N 首」 |
| 列表项 | 封面缩略图（圆角方形，48dp）、歌曲名（单行，超长省略）、歌手名（单行，辅助色） |
| 排序规则 | 按收藏时间倒序（最新收藏在最上方） |
| 空状态 | 居中展示空状态插图 + 文案「还没有收藏歌曲，去首页搜索喜欢的歌曲吧」 |

**交互行为**

| 操作 | 行为 |
|------|------|
| 点击列表项 | 播放该歌曲，同时将整个收藏列表设为当前播放歌单 |
| 点击「播放全部」 | 从第一首开始播放整个收藏列表 |
| 左滑列表项 | 露出「取消收藏」按钮，点击后移除该歌曲（无需二次确认） |
| 在详情页取消收藏 | 返回后列表实时更新 |

#### 3.2.4 验收标准

- [ ] 收藏列表按时间倒序正确展示
- [ ] 点击歌曲可播放，并将收藏列表设为当前歌单
- [ ] 「播放全部」从第一首开始播放
- [ ] 可左滑取消收藏，列表实时更新
- [ ] 空状态有引导提示

### F3. 播放历史

**优先级**：P0 | **复杂度**：中

#### 3.3.1 需求描述

自动记录用户的歌曲播放行为，在「我的」页面的「播放历史」Tab 中展示，帮助用户回溯和重新发现歌曲。

#### 3.3.2 记录规则

| 规则 | 说明 |
|------|------|
| **触发时机** | 歌曲开始播放时自动记录（无论来源：搜索、收藏、历史） |
| **去重策略** | 同一首歌（相同 threadId）仅保留最新一条记录，更新时间戳并置顶 |
| **存储上限** | 200 条 |
| **淘汰策略** | 超过上限时，自动删除最早的记录 |
| **持久化** | SharedPreferences，JSON 序列化（与 FavoritesManager 方案一致） |

#### 3.3.3 存储字段

| 字段 | 类型 | 说明 |
|------|------|------|
| threadId | String | 歌曲唯一标识 |
| title | String | 歌曲名 |
| artist | String | 歌手名 |
| coverUrl | String | 封面图 URL |
| audioUrl | String | 音频 URL（用于直接播放，无需重新加载详情） |
| playedAt | Long | 播放时间戳（毫秒） |

#### 3.3.4 播放历史 Tab

**列表展示**

| 元素 | 说明 |
|------|------|
| 操作栏 | 左侧「播放全部」按钮，右侧显示歌曲总数「共 N 首」 |
| 列表项 | 与收藏列表一致：封面、歌曲名、歌手名 |
| 排序规则 | 按播放时间倒序（最近播放在最上方） |
| 空状态 | 居中展示空状态插图 + 文案「还没有播放记录」 |

**交互行为**

| 操作 | 行为 |
|------|------|
| 点击列表项 | 播放该歌曲，同时将整个播放历史列表设为当前播放歌单 |
| 点击「播放全部」 | 从第一首开始播放整个播放历史列表 |
| 清空历史 | TopAppBar 右侧提供「清空」按钮，点击后弹出确认弹窗，确认后清空全部记录 |

#### 3.3.5 验收标准

- [ ] 播放歌曲时自动记录到播放历史
- [ ] 重复播放同一首歌更新时间戳，不产生重复记录
- [ ] 历史列表按时间倒序展示
- [ ] 超过 200 条时自动淘汰最早记录
- [ ] 点击歌曲可播放，并将历史列表设为歌单
- [ ] 「播放全部」从第一首开始播放
- [ ] 「清空」操作有二次确认，确认后清空
- [ ] 空状态有提示文案

### F4. 歌单播放（播放队列）

**优先级**：P0 | **复杂度**：高

#### 3.4.1 需求描述

引入「当前播放歌单」概念。用户从不同入口触发播放时，自动建立对应的播放歌单。播放器的上一首 / 下一首操作在歌单范围内执行。

#### 3.4.2 歌单建立规则

| 触发场景 | 歌单内容 | 当前播放位置 |
|----------|----------|-------------|
| 收藏列表 — 点击某首歌 | 整个收藏列表 | 点击的那首歌 |
| 收藏列表 — 播放全部 | 整个收藏列表 | 第一首 |
| 播放历史 — 点击某首歌 | 整个播放历史列表 | 点击的那首歌 |
| 播放历史 — 播放全部 | 整个播放历史列表 | 第一首 |
| 搜索结果 — 点击进入详情后播放 | 仅当前单曲 | 当前歌曲 |

#### 3.4.3 切歌逻辑

**上一首 / 下一首**

```
                    ┌──────────────────────────┐
                    │      当前播放歌单          │
                    │  [歌曲A] [歌曲B] [歌曲C]   │
                    │              ↑             │
                    │           当前播放          │
                    │     ← 上一首   下一首 →     │
                    └──────────────────────────┘
```

| 操作 | 默认行为（顺序播放模式） |
|------|------------------------|
| 点击「上一首」 | 切换到歌单中前一首。如在第一首，循环到最后一首 |
| 点击「下一首」 | 切换到歌单中后一首。如在最后一首，循环到第一首 |
| 当前歌曲播放完毕 | 自动播放下一首。如在最后一首，循环到第一首继续播放 |

**单曲模式**

当歌单仅包含一首歌曲时（来自搜索结果的单曲播放）：
- 上一首 / 下一首按钮置灰不可用
- 播放完毕后自动停止（除非设为单曲循环）

#### 3.4.4 歌单与播放器状态

播放器需对外暴露以下歌单相关状态，供 UI 消费：

| 状态字段 | 类型 | 用途 |
|----------|------|------|
| currentPlaylist | List\<SongInfo\> | 当前歌单内容 |
| playlistSource | enum | 歌单来源（FAVORITES / HISTORY / SINGLE） |
| currentIndex | Int | 当前播放位置 |
| hasNext | Boolean | 是否有下一首 |
| hasPrevious | Boolean | 是否有上一首 |

#### 3.4.5 验收标准

- [ ] 从收藏列表触发播放时，整个收藏列表被设为当前歌单
- [ ] 从播放历史触发播放时，整个历史列表被设为当前歌单
- [ ] 从搜索详情触发播放时，歌单仅包含当前单曲
- [ ] 上一首 / 下一首在歌单范围内正确切歌
- [ ] 歌单首尾可循环
- [ ] 单曲歌单下，上一首 / 下一首按钮不可用
- [ ] 歌曲播放完毕自动播放下一首
- [ ] 切歌后 Mini 播放器信息同步更新

### F5. 播放模式

**优先级**：P1 | **复杂度**：中

#### 3.5.1 需求描述

支持三种播放模式，用户可在播放时自由切换，满足不同场景的收听偏好。

#### 3.5.2 模式定义

| 模式 | 图标 | 下一首行为 | 播放完毕行为 |
|------|------|-----------|-------------|
| 顺序播放 | `Icons.Default.Repeat`（列表循环） | 歌单中下一首 | 循环到第一首继续播放 |
| 随机播放 | `Icons.Default.Shuffle` | 歌单中随机一首（不重复） | 全部播放完后重新打乱 |
| 单曲循环 | `Icons.Default.RepeatOne` | 歌单中下一首 | 重复播放当前歌曲 |

**随机播放补充规则**：
- 打乱歌单中未播放的歌曲顺序
- 在所有歌曲播放一遍之前不会重复
- 全部播放完后重新打乱顺序，继续循环

**单曲循环补充规则**：
- 当前歌曲播放完毕后自动重新播放
- 手动点击「上一首 / 下一首」仍然在歌单内切歌（不受单曲循环限制）

#### 3.5.3 交互设计

| 交互 | 说明 |
|------|------|
| **入口位置** | 详情页播放器控制区域，位于上一首按钮左侧 |
| **切换方式** | 点击按钮，在三种模式间循环：顺序 → 随机 → 单曲循环 → 顺序 |
| **反馈** | 切换后图标变化 + Toast 提示模式名称（如「随机播放」） |
| **持久化** | 用户选择的播放模式在 App 重启后保留 |
| **默认值** | 新安装默认「顺序播放」 |

#### 3.5.4 验收标准

- [ ] 播放器控制区域显示播放模式按钮
- [ ] 点击按钮在三种模式间正确循环切换
- [ ] 顺序播放模式下按歌单顺序播放
- [ ] 随机播放模式下不重复播放，全部播完后重新打乱
- [ ] 单曲循环模式下当前歌曲自动重复
- [ ] 单曲循环模式下手动切歌仍可切到上一首 / 下一首
- [ ] 播放模式 App 重启后保持
- [ ] 切换模式时显示 Toast 提示

## 4. 交互流程

### 4.1 核心用户旅程：从收藏列表连续听歌

```
用户打开 App
    │
    ├─ 默认进入「首页」Tab
    │
    ├─ 点击底部导航栏「我的」Tab
    │     │
    │     ├─ 默认展示「收藏歌曲」列表
    │     │
    │     ├─ 点击「播放全部」
    │     │     │
    │     │     ├─ 从第一首开始播放
    │     │     ├─ 整个收藏列表设为当前歌单
    │     │     └─ Mini 播放器出现，显示当前歌曲信息
    │     │
    │     └─ 或点击某首歌
    │           │
    │           ├─ 播放该歌曲
    │           ├─ 整个收藏列表设为当前歌单（定位到该歌曲）
    │           └─ Mini 播放器出现
    │
    ├─ 点击 Mini 播放器
    │     │
    │     └─ 进入歌曲详情页
    │           ├─ 底部导航栏隐藏
    │           ├─ 显示完整播放器控制（上一首、播放/暂停、下一首、播放模式）
    │           ├─ 点击「下一首」→ 播放歌单中下一首
    │           └─ 返回 → 恢复「我的」页面，导航栏重新显示
    │
    └─ 歌曲播放完毕 → 根据播放模式自动播放下一首
```

### 4.2 播放历史记录时序

```
用户触发播放（任意来源）
    │
    ├─ MusicPlayerManager.play(songInfo)
    │     │
    │     ├─ ExoPlayer 开始播放
    │     │
    │     └─ 同时：PlayHistoryManager.record(songInfo)
    │           │
    │           ├─ 检查是否已存在相同 threadId 的记录
    │           │     ├─ 是 → 更新 playedAt 时间戳
    │           │     └─ 否 → 新增记录
    │           │
    │           ├─ 检查总数是否超过 200
    │           │     └─ 是 → 删除最早的记录
    │           │
    │           └─ 持久化到 SharedPreferences
    │
    └─ 「我的 - 播放历史」列表自动刷新（如果页面可见）
```

## 5. 非功能需求

| 类别 | 要求 | 说明 |
|------|------|------|
| **性能** | 列表加载 < 200ms | 收藏 / 历史列表本地数据加载 |
| **存储** | 增量约 100KB | 200 条历史记录的 JSON 存储开销 |
| **兼容性** | Android 8.0+ (API 26) | 与现有最低版本一致 |
| **稳定性** | 播放器状态不丢失 | Tab 切换、页面跳转不影响当前播放 |
| **离线** | 收藏和历史可离线浏览 | 数据存储在本地，不依赖网络 |
| **无障碍** | 按钮有 contentDescription | 播放模式等按钮需有语义描述 |

## 6. 影响范围与技术评估

### 6.1 现有模块变更

| 模块 | 变更类型 | 变更内容 |
|------|----------|----------|
| `MainActivity` | **重构** | Scaffold 布局改造：新增 `BottomNavigationBar`，将 `MiniPlayerBar` 从 bottomBar 移至 Column 布局中导航栏上方 |
| `MusicPlayerManager` | **扩展** | 新增歌单管理能力：`setPlaylist(songs, startIndex)`、播放模式状态与切换逻辑、扩展 `PlayerState` |

## 附录：术语表

| 术语 | 定义 |
|------|------|
| **歌单** | 一组有序的歌曲集合，用户播放操作（上一首/下一首）的上下文范围 |
| **播放队列** | 等同于歌单，指当前播放器正在消费的歌曲列表 |
| **播放模式** | 控制歌单内歌曲的播放顺序和循环方式 |
| **歌单来源** | 歌单建立的入口，如收藏列表、播放历史、搜索结果等 |
| **Mini 播放器** | 底部悬浮的简化播放器，展示当前播放歌曲信息和基本控制 |
