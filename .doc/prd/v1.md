## 一、背景与目标

### 1.1 背景

HiFiTi Android App 当前已具备搜索、播放、收藏、下载等核心功能，但存在以下体验痛点：

- **导航结构缺失**：当前页面通过 NavHost 路由跳转，无底部导航栏，用户无法快速切换核心模块
- **播放历史缺失**：用户无法回溯已播放过的歌曲，不利于歌曲的再次发现
- **歌单播放能力不足**：当前播放器仅支持单曲级别操作，缺乏歌单维度的连续播放、切歌和播放模式控制

### 1.2 目标

本次迭代目标为：**构建完善的导航架构、歌单播放体系和播放历史能力**，具体包括：

1. 建立底部导航栏，提供「首页」和「我的」两个常驻入口
2. 新增播放历史功能，记录用户播放行为
3. 实现歌单维度的连续播放（收藏列表 / 播放历史）
4. 支持顺序播放、随机播放、单曲循环三种播放模式

## 三、功能需求

### 3.1 底部导航栏

**优先级**：P0

#### 3.1.1 功能描述

在 App 底部添加常驻导航栏，包含两个 Tab：

| Tab | 图标 | 标签 | 对应页面 |
|-----|------|------|----------|
| 首页 | Home 图标 | 首页 | 搜索页（当前首页） |
| 我的 | Person 图标 | 我的 | 个人中心页（新增） |

#### 3.1.2 交互规则

- 导航栏始终显示在屏幕底部，Mini 播放器位于导航栏上方
- 点击 Tab 切换页面，当前选中 Tab 高亮
- 切换 Tab 时保持各页面状态（如搜索结果、滚动位置）
- 从详情页返回时，恢复到之前选中的 Tab
- 详情页中不显示底部导航栏（仅显示 Mini 播放器）

#### 3.1.3 层级关系

```
┌──────────────────────────────┐
│         页面内容区域           │
│   (首页搜索 / 我的 / 详情)     │
├──────────────────────────────┤
│       Mini 播放器栏           │  ← 有歌曲播放时显示
├──────────────────────────────┤
│    🏠 首页    |    👤 我的     │  ← 底部导航栏（详情页隐藏）
└──────────────────────────────┘
```

### 3.2「我的」页面

**优先级**：P0

#### 3.2.1 页面结构

「我的」页面分为两个 Tab 子页签：

| 子 Tab | 内容 |
|--------|------|
| 收藏歌曲 | 用户收藏的歌曲列表 |
| 播放历史 | 用户播放过的歌曲列表 |

#### 3.2.2 收藏歌曲 Tab

- 展示用户所有已收藏的歌曲，按收藏时间倒序
- 每行展示：封面缩略图、歌曲名、歌手名
- 列表顶部显示「播放全部」按钮 + 歌曲总数
- 点击歌曲行 → 直接播放该歌曲，并将整个收藏列表设为当前歌单
- 左滑/长按 → 取消收藏
- 列表为空时展示空状态引导

#### 3.2.3 播放历史 Tab

- 展示用户播放过的歌曲记录，按播放时间倒序
- 每行展示：封面缩略图、歌曲名、歌手名
- 列表顶部显示「播放全部」按钮 + 歌曲总数
- 点击歌曲行 → 直接播放该歌曲，并将整个播放历史列表设为当前歌单
- 支持「清空播放历史」操作（需二次确认）
- 历史记录上限：200 条，超出时自动淘汰最早的记录
- 同一首歌重复播放时，更新时间戳并移至列表顶部（去重）
- 列表为空时展示空状态引导

### 3.3 播放历史记录

**优先级**：P0

#### 3.3.1 功能描述

自动记录用户的歌曲播放行为，持久化存储。

#### 3.3.2 记录规则

- **触发时机**：歌曲开始播放时记录（不论来源：搜索结果、收藏、历史）
- **存储字段**：threadId、歌曲名、歌手名、封面URL、播放时间戳
- **去重规则**：同一 threadId 仅保留最新一条记录
- **存储上限**：200 条
- **淘汰策略**：超出上限时删除最早的记录
- **持久化方式**：SharedPreferences（与现有 FavoritesManager 保持一致）

---

### 3.4 歌单播放

**优先级**：P0

#### 3.4.1 功能描述

引入「当前歌单」概念，播放器的上一首/下一首操作基于歌单上下文执行。

#### 3.4.2 歌单来源

| 来源 | 触发方式 | 歌单内容 |
|------|----------|----------|
| 收藏列表 | 在「我的 - 收藏歌曲」中点击播放或点击「播放全部」 | 当前收藏列表的所有歌曲 |
| 播放历史 | 在「我的 - 播放历史」中点击播放或点击「播放全部」 | 当前播放历史的所有歌曲 |
| 搜索结果 | 在搜索结果中点击播放歌曲 | 仅包含当前播放的单曲（单曲模式） |

#### 3.4.3 切歌逻辑

- **上一首**：播放当前歌单中的上一首歌曲
  - 到达第一首时：循环到最后一首（顺序/随机模式）
- **下一首**：播放当前歌单中的下一首歌曲
  - 到达最后一首时：循环到第一首（顺序/随机模式）
- **自动下一首**：当前歌曲播放完毕后，根据播放模式自动播放下一首
- 歌单为单曲时，上一首/下一首不可用（灰态）

---

### 3.5 播放模式

**优先级**：P1

#### 3.5.1 功能描述

支持三种播放模式，用户可自由切换。

| 模式 | 图标 | 行为说明 |
|------|------|----------|
| 顺序播放 | 🔁 列表循环图标 | 按歌单顺序依次播放，末尾循环到开头 |
| 随机播放 | 🔀 随机图标 | 随机选择歌单中的下一首（不重复，直到全部播放一遍） |
| 单曲循环 | 🔂 单曲循环图标 | 当前歌曲结束后重新播放 |

#### 3.5.2 交互规则

- 播放模式切换按钮位于播放器控制区域
- 点击按钮在三种模式间循环切换：顺序 → 随机 → 单曲循环 → 顺序
- 切换时显示 Toast 提示当前模式名称
- 播放模式持久化存储，App 重启后保持上次设置

## 四、非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 收藏/历史列表加载时间 < 200ms |
| 存储 | 播放历史采用 SharedPreferences，与现有方案一致 |
| 兼容性 | 最低支持 Android API 26 (Android 8.0) |
| 状态恢复 | Tab 切换不丢失页面状态 |
| 离线 | 收藏列表和播放历史在离线状态下可正常浏览 |

### 5.1 新增：PlayHistoryItem

```kotlin
data class PlayHistoryItem(
    val threadId: String,
    val songName: String,
    val artist: String,
    val coverUrl: String,
    val playedAt: Long  // 时间戳
```

### 5.2 新增：Playlist

```kotlin
data class Playlist(
    val source: PlaylistSource,  // FAVORITES / HISTORY / SINGLE
    val songs: List<SongInfo>,
    val currentIndex: Int
)

enum class PlaylistSource {
    FAVORITES, HISTORY, SINGLE
}
```

### 5.3 新增：PlayMode

```kotlin
enum class PlayMode {
    SEQUENTIAL,   // 顺序播放
    SHUFFLE,      // 随机播放
    REPEAT_ONE    // 单曲循环
}
```
